Comparing Windows, macOS, and Linux Operating Systems

Files and Directories, Naming Conventions, and the File Allocation Table (FAT)

1.1. Directory Structure

  • Windows: Uses a drive-based structure (e.g., C:\Users\Vincent\Documents)
    Drives like C:\ represent physical or logical storage.
  • macOS: Uses a UNIX-like structure (e.g., /Users/john/Documents), beginning at the root /.
  • Linux: Also uses a UNIX-like structure, starting at / and branching into /home, /etc, /usr, etc

1.2. File Paths

  • Windows: Uses backslashes \ ‚Üí C:\Program Files\App
  • macOS & Linux: Use forward slashes / ‚Üí /usr/bin/app

2. Naming Conventions

  • Windows: Not case-sensitive, disallows \ / : * ? " < > |
  • macOS: Typically not case-sensitive but can be configured to be; avoids ":" in names :
  • Linux: Case-sensitive; disallows / and null character

3. File Systems

  • Windows: NTFS, FAT32, exFAT
  • macOS: APFS, HFS+, FAT32
  • Linux: ext4; supports FAT32
  • FAT: File Allocation Table ‚Äì still used in USB drives/SD cards for cross-OS compatibility

Executable File Extensions

1. Executable File Types

  • Windows: .exe, .bat, .msi, .cmd
  • macOS: No extension required, but apps are .app bundles
  • Linux: No extension required; executability defined by permissions

2. Script Execution

  • Windows: .bat for batch scripts, .ps1 via PowerShell or CMD
  • macOS: .sh or no extension shell scripts, run via Terminal using bash/zsh
  • Linux: .sh, .bash, .py, executable with chmod +x
